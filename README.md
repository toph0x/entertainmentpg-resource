# entertainmentpg-resource

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以资源形式存储，不针对任何特定域名或具体网站内容。

## 项目简介

这是一个以 HTML 页面为主的资源仓库，主要目的为：

- 集中存放各类独立页面
- 便于版本管理与历史追溯
- 提供一种简单的发布与访问方式

每个页面均为独立文件，彼此之间无强关联，可根据需要单独引用或下载。

## 目录说明

- `pages/` — 存放主要 HTML 页面文件
- `assets/` — 页面引用的静态资源（如 CSS、JavaScript、图片等）
- `docs/` — 相关说明文档或附加信息（可选）

仓库根目录下的 `index.html` 可作为导航页或入口页面，具体内容视提交情况而定。

## 页面归档说明

所有页面均以时间或功能分类存放，命名规则建议采用以下格式：

```
[类别]-[功能描述]-[版本].html
```

例如：

```
tool-calendar-v1.html
game-snake-v2.html
```

如无特殊说明，每个页面均可独立运行，无需额外环境配置。资源文件路径相对页面位置固定，请勿随意移动。

## 维护说明

- 本仓库由维护者不定期更新
- 欢迎通过 Issue 提交建议或报告问题
- 提交 Pull Request 前请确保页面功能正常，且不包含外部依赖（除非在 assets 内提供）
- 不建议在页面中加入跟踪、统计或第三方服务代码，以保持仓库的通用性

## 使用方式

1. 克隆或下载仓库到本地
2. 直接打开 `pages/` 目录下的 HTML 文件即可查看
3. 若需部署，可将整个仓库内容上传至静态服务器
